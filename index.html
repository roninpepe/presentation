Coordinates

For all elements, SVG uses a coordinate system similar to the one used by HTML or canvas. That is, the top left corner of the document is considered to be the point (0,0), or point of origin. Positions are then measured in pixels from the top left corner, with the positive x direction being to the right, and the positive y direction being to the bottom.

viewBox

The value of the viewBox attribute is a list of four numbers: min-x, min-y, width and height. The numbers separated by whitespace and/or a comma, which specify a rectangle in user space which is mapped to the bounds of the viewport established for the associated SVG element.

Scaling

With a viewBox correctly set on the image, it will scale to the dimensions of it’s container.

Basic shapes

There are several basic shapes used for most SVG drawing. The purpose of these shapes is fairly obvious from their names.

  • rect
  • circle
  • ellipse
  • line
  • polyline
  • polygon
  • path

rect

The <rect> element draws a rectangle on the screen. There are 6 basic attributes that control the position and shape of the rectangles on screen. The one on the bottom has its rx and ry parameters set, giving it rounded corners. If they're not set, they default to 0.

circle

The <circle> element draws a circle on the screen. It takes 3 basic parameters to determine the shape and size of the element.

ellipse

An <ellipse> is a more general form of the <circle> element, where you can scale the x and y radius (commonly referred to as the semimajor and semiminor axes in maths) of the circle separately.

line

The <line> element takes the positions of two points as parameters and draws a straight line between them.

polyline

polygon

A <polygon> is similar to a <polyline>, in that it is composed of straight line segments connecting a list of points. For polygons though, the path automatically connects the last point with the first, creating a closed shape.

path

A <path> is the most general shape that can be used in SVG. Using a path element, you can draw rectangles (with or without rounded corners), circles, ellipses, polylines, and polygons. Basically any of the other types of shapes, bezier curves, quadratic curves, and many more.

Accessibility

You can make your SVG accessible by adding screen readers support or keyboard-only navigation. But you can use some tricks to make your UI more user-friendly for all users.

<title>

The <title> SVG tag similar to an <img> title attribure. The <title> must be the first child of it’s parent element and will be used as a tooltip as the pointing device moves over it.

<desc>

The <desc> SVG tag can contain a longer, more complete description of image than a <title>. This text will not be displayed on the screen.

aria-labelledby

To ensure the assistive technology can access the <title> and <desc>:

  • Add the ID’s to the <title> and <desc>:
    • <title id="uniqueTitleID">…</title>
    • <desc id="uniqueDescID">…</desc>
  • On the <svg> tag, add:
    • aria-labelledby="uniqueTitleID uniqueDescID" – both title and description are included in aria-labelledby because it has better screen-reader support than aria-describedby

<text>

The <text> SVG tag draws a graphics element consisting of text. It could be used as a <title> and a <desc> for NVDA (not Nvidia) AT. Also it could be used as <img> alt attribure.

tabindex

You can make SVG elements focusable by keyboard via adding tabindex attribute.
